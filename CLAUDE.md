# legalsol.pl — правила работы с репо

## ⚠️ ВАЖНО с 2026-07-08: JS главной вынесен в бандл

Почти весь JS главной живёт НЕ в `_index.html`, а в `/assets/app-<hash>.js`
(defer), собираемом из inline-блоков скриптом **`scripts/extract_js.py`**.

**Как править JS главной страницы:**
1. НЕ добавляй новые большие inline `<script>` в `_index.html` — клади логику так,
   чтобы её подхватил экстрактор, либо правь и запускай пересборку.
2. После ЛЮБОЙ правки inline-JS в `_index.html`: `python3 scripts/extract_js.py`
   (пересоберёт бандл с новым hash и перепишет ссылку). Коммить и `_index.html`,
   и `assets/app-*.js` вместе.
3. Остаются inline и НЕ выносятся: ранний lng-init гейт, consent-стаб, GTM-деферер
   (`__gtmGo`), `_lsGtag`, IO-гидратор (`data-src`/`data-bg`), клик-шим
   (`__earlyClickQ`). Список — в KEEP_MARKS экстрактора.
4. Семантика бандла: блоки в try/catch (изоляция ошибок), init главного блока
   отложен setTimeout(0). Sync-код на верхнем уровне видит ПОЛНЫЙ DOM (не
   частичный, как при parse-time) — гардь idempotency при повторных прогонах.

## Прочее
- `_index.html` минифицирован — искать по якорям/id, не по номерам строк.
- База разметки — АНГЛИЙСКИЙ (с 2026-07-07). Переводы: главный словарь `T`
  (en+ru inline), остальные языки лениво из `/lang/*.json`.
- Деплой: push в main → Vercel (прод сразу). Параллельные сессии пушат в main —
  перед merge: `git fetch` + пересборка на свежем main; мержить быстро.
- Перф-статус и историю оптимизаций см. в памяти `legalsol-site-perf-2026-06`.
- `middleware.js` НЕ удалять (закрывает source-disclosure).
